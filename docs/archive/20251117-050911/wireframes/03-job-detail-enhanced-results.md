## Key Interaction Patterns

### Expandable Row Behavior

1. **Initial State**: All rows collapsed, showing summary info
2. **Click [>] or row**: Expand to show full breakdown
3. **Expanded State**:
   - Row expands vertically with smooth animation
   - Shows complete Layer 1/2/3 analysis in sections
   - Each layer clearly marked with status (PASSED/REJECTED/ACCEPTED)
   - Icon changes to [â–¼] indicating expanded state
4. **Click [â–¼]**: Collapse back to summary view

### Filter Behavior

- **Classification Filter**: Show only Accept or Reject decisions
- **Decision Layer Filter**: Show only URLs decided at specific layer (L1/L2/L3)
- **Confidence Filter**: Only applies to Layer 3 URLs (others show N/A)
- **Search**: Fuzzy match against URL text
- **Combine Filters**: All filters work together (AND logic)
- **Active Filter Badges**: Show which filters are active above table
- **Clear All**: Button to reset all filters

### Confidence Indicators

- **ğŸŸ¢ High (0.8-1.0)**: Green dot, high confidence in classification
- **ğŸŸ¡ Medium (0.6-0.8)**: Yellow dot, moderate confidence - may need review
- **ğŸŸ  Low (0.4-0.6)**: Orange dot, low confidence - recommend manual review
- **ğŸ”´ Very Low (<0.4)**: Red dot, very uncertain - definitely review
- **N/A**: For Layer 1/2 eliminations (no LLM confidence score)

### Cost Display

- **Layer 1 URLs**: $0.00 (rule-based, free)
- **Layer 2 URLs**: $0.00 (heuristic, free)
- **Layer 3 URLs**: Actual LLM cost per URL (varies by token count)

    rejectionReason?: string;
    processingTime: number;

  // Layer 3 Data (if processed)

## User Journey

1. **User uploads CSV with 1,250 URLs**
2. **Job processes through 3 layers** (shown in Overview tab)
3. **User navigates to Results tab** - sees 342 accepted URLs
4. **User filters by "Medium confidence"** - finds 112 URLs to review
5. **User expands first URL** - reads full Layer 3 reasoning
6. **User understands WHY** - sees feature article vs. news article distinction
7. **User adjusts classification prompt** - notes for next job
8. **User exports "Detailed Results"** - gets CSV with analysis summaries
9. **User reviews exported data** - makes final decisions in spreadsheet

---

## Design Philosophy

This page embodies **transparency and education**:

- Every decision is explainable
- Every layer's logic is visible
- Users learn what works and what doesn't
- No black box classification
- Confidence scores guide manual review
- Export preserves all context for downstream analysis

The expandable row is the **key innovation** - users can drill into any URL to understand the complete decision path from domain check through LLM reasoning.
